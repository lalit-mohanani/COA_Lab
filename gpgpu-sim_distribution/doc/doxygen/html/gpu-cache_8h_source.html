591  // For more details about difference between FETCH_ON_WRITE and WRITE
592  // VALIDAE policies Read: Jouppi, Norman P. "Cache write policies and
593  // performance". ISCA 93. WRITE_ALLOCATE is the old write policy in
594  // GPGPU-sim 3.x, that send WRITE and READ for every write request
595  switch (wap) {
596  case 'N':

598  break;

612  // detect invalid configuration
613  if (m_alloc_policy == ON_FILL and m_write_policy == WRITE_BACK) {
614  // A writeback cache with allocate-on-fill policy will inevitably lead to
615  // deadlock: The deadlock happens when an incoming cache-fill evicts a
616  // dirty line, generating a writeback request. If the memory subsystem is
617  // congested, the interconnection network may not have sufficient buffer
618  // for the writeback request. This stalls the incoming cache-fill. The
619  // stall may propagate through the memory subsystem back to the output
620  // port of the same core, creating a deadlock where the wrtieback request
621  // and the incoming cache-fill are stalling each other.
622  assert(0 &&
623  "Invalid cache configuration: Writeback cache cannot allocate new "
624  "line on fill. ");
625  }
